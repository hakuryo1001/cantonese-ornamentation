# cantonese-pgfornament

LaTeX runtime package for Cantonese ornaments. Extends [pgfornament](https://ctan.org/pkg/pgfornament) with a `cantonese` ornament family.

## Layout

```text
cantonese-pgfornament/
├── generic/cantonese/    # copied .pgf + .dim.json assets
├── latex/                # hand-written + generated runtime code
│   ├── cantoneseornament.sty
│   ├── cantonese-frame.sty
│   ├── sinoglyph.sty
│   └── pgflibrarycantonese.code.tex   # generated by gen-library.py
└── tools/
    └── gen-library.py    # rebuild dimension table from copied assets
```

SVG sources and the conversion pipeline live in the separate **`canto-pfg-generator`** repo.

PGF ornaments are geometry-only silhouettes. Set colour (including forcing black for print) at render time:

```latex
\cantoneseornament[width=2cm, color=black]{1}
\cantoneseornament[width=2cm, color=Maroon]{1}
```

## Usage

```latex
\usepackage[object=pgfhan]{pgfornament}
\usepackage{cantoneseornament}

\cantoneseornament[width=2cm, color=Maroon]{1}
\cantoneseframe[corner=1, edge=16, width=8cm]{Content}
\cantonesedivider{77}
```

## Receiving new assets

When SVG designs are converted in `canto-pfg-generator`, copy outputs manually:

```bash
cp /path/to/canto-pfg-generator/generated/cantonese/*.pgf generic/cantonese/
cp /path/to/canto-pfg-generator/generated/cantonese/*.dim.json generic/cantonese/
# For full-colour display, also copy source PNGs (rename to flora{id}.png):
cp /path/to/canto-pfg-generator/source/flora/banyan.png assets/flora/flora101.png
# ... etc.
python3 tools/gen-library.py
```

Full-colour flora ornaments use `\cantonesefloraornament{width=2cm}{101}` (PNG in
`assets/flora/`). Monochrome vector silhouettes use `\cantoneseornament{color=…}{101}` (PGF).

Then recompile the catalog in the parent `cantonese-ornamentation` project.

## What not to edit by hand

- `generic/cantonese/*.pgf` — generated from SVGs
- `generic/cantonese/*.dim.json` — generated sidecars
- `latex/pgflibrarycantonese.code.tex` — run `tools/gen-library.py` instead
